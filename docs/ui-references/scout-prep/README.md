# Scout Prep UI References

## Direction

Scout Prep should feel like a focused Mac task-detail surface:

- Raycast speed and command orientation.
- Asana-style task list plus selected task details pane.
- Prospect ID admin CRM field grouping and spacing.

## Use

- Clear selected-athlete header.
- Current task, due state, status/stage chips.
- Dense but calm athlete/contact/scouting fields.
- Organized action panel/menu behavior instead of a wall of buttons.
- Notes/activity below the primary task context.

## Avoid

- Visible function names.
- Adapter/debug copy.
- Marketing copy.
- Repeating every workflow just because the screen has space.

## Asana Translation Rules

Use Asana only for layout and visual hierarchy:

- Main pane + selected task detail pane, not new workflow objects.
- Sections/grouping for readable task detail areas, not new task statuses.
- Task colors based on existing Scout Prep task kind/title/due state, not Asana custom fields.
- Compact action controls grouped like a task detail action area, but labels remain Prospect Pipeline labels.
- Activity/notes below task context, not a second CRM timeline.

Do not import Asana concepts such as projects, collaborators, approvals, goals, portfolios, automations, or custom-field business meaning into Scout Prep.
