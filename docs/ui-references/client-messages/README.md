# Client Messages UI References

## Direction

Client Messages should stay a side-panel workflow unless directly opened from Scout Prep.

## Use

- Inbox-like rows.
- Recipient clarity.
- Message context and last-touch timing.
- Fast copy/send/open actions.

## Avoid

- Turning messages into a parallel CRM.
- Showing internal send/verification function names.
- Adding message workflows outside Prospect Pipeline/Raycast boundaries.
