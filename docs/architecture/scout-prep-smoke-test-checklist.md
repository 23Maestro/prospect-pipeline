# Scout Prep Smoke Test Checklist

Use this after architecture, owner, or reporting changes that affect Scout Prep, Head Scout Schedules, View Set Meetings, FastAPI, or Supabase.

1. Open Scout Prep command.
2. Run post-call update for a normal voicemail/contact stage.
3. Confirm Laravel sales stage update still persists.
4. Set a meeting from Raycast.
5. Confirm Laravel/Prospect ID meeting is created.
6. Confirm email/text workflow still fires.
7. Open View Set Meetings.
8. Confirm only active-operator meetings show.
9. Send confirmation 1.
10. Send confirmation 2.
11. Confirm phrasing says this afternoon / tonight / tomorrow morning correctly.
12. Confirm Supabase Call Tracker reports only materialized rows.
13. Confirm Tim or other coordinator meetings do not appear as Jerami-owned.
14. Confirm Scout Openings still lists Jeffrey/Luther/Ryan/James open slots.
15. Confirm Head Scout calendar owner IDs still match legacy behavior.
16. Confirm FastAPI legacy adapter routes still return expected data.
